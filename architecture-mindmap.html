🏢 办公室模拟 - 架构思维导图

Godot 本地状态机 + 后端 Atmosphere Service | 重构后 v2.0

一、WorkerStateMachine 四状态有限状态机
⏸ IDLE 待命/断网
💻 WORKING 在工位工作
☕ BREAK 休息区放松
🚶 ROAMING 闲逛走动
有任务 && 在工位上
energy<0.35 && P<15%
无任务 && P<8%
energy>0.85 或 定时到期
到达工位 或 有新任务
force_idle() 断网时

全局约束：shared_away_count <= 3（所有状态机实例共享的静态变量）
能量衰减：WORKING energy-=0.008/s stress+=0.003/s | BREAK energy+=0.04/s stress-=0.02/s | ROAMING energy-=0.004/s
tick() 返回：TickResult(target_type: "own_desk"/"idle_point"/"stay", state_changed: bool)

四、两层记忆系统（MemGPT/Letta 风格）
始终在上下文中
Core Memory (~400字)
• 身份：名字、岗位
• 性格：personality 描述
• 工作方式 / 沟通风格
• 重视事项 work_values
• 来源：agent profile 直接生成
• 不读磁盘文件
按需加载
Working Memory (~8条)
• 最近有价值的事件
• 老板指令（提到自己的）
• 任务变更 / 完成
• 协作确认 / 风险发现
• 来源：内存列表取最近 N 条
• 过滤了闲聊/移动/idle
不参与 LLM 上下文
Archival (events.jsonl)
• 完整事件流水日志
• 继续写入但不读取给 LLM
• 仅用于调试/人工查看
• 写入前过滤低价值内容
